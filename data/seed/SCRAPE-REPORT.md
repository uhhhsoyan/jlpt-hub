## MAJOR DEVIATION: N4 vocab list is not actually cumulative

The task brief describes the N4 vocab source page as cumulative (including all N5 words). As currently published, it is **not**. It contains only 677 rows (vs the ~1,500 expected), and is missing large numbers of basic N5 words that are clearly N4-or-below level -- e.g. 食べる (たべる, to eat), 先生 (せんせい, teacher), 見る (みる, to see), 読む (よむ, to read), 学校 (がっこう, school) all appear in vocab-n5.json but NOT in vocab-n4.json.

This was verified to not be a fetch artifact: re-fetched with three different User-Agents/header sets (identical bytes each time) and cross-checked against a Wayback Machine snapshot from ~4 months before this scrape (2026-03-03), which has the exact same row count (761 `<tr>`, 42 tables) and is missing the same words. The site's N4 vocab page itself has not been cumulative for some time.

Per the task instructions this was NOT fixed by merging N5 into N4 -- vocab-n4.json contains exactly what nihongoichiban.com's N4 page currently has. The downstream `scripts/seed-items.mjs` dedup step assumes vocab-n4.json is a cumulative superset of vocab-n5.json; with the source no longer cumulative, most N5 words will not be found as overlaps and the two levels will need reconciling upstream of (or within) that seed step.
